## ⚡ 一句话速览

> OpenStory 是浙大 ZJU-LLMs 团队开源的多智能体推演框架，基于 LLM+Agent-Kernel，首个官方实现为《红楼梦》大观园 1:1 可视化模拟，支持动态增删 Agent，Apache-2.0 开源。

- **官网地址**：https://github.com/ZJU-LLMs/OpenStory

## 2. 核心功能

- **动态 Agent-Kernel 框架**⭐：支持运行时动态增删 Agent，无需重启系统，赋予系统无限可扩展性
- **1:1 大观园可视化前端**⭐：精细还原的红楼梦地图，实时显示角色位置、动作、对话和记忆状态
- **多智能体协同推理**：基于 LLM，每个 Agent 有独立感知-规划-执行-反思生命周期
- **故事推演生成**：根据角色性格和环境变化，自动生成有戏剧张力的推演故事
- **YAML 高配置化**：通过 YAML 文件灵活管理系统、环境、动作和 Agent 配置
- **语音功能**：支持语音交互（新增中）
- **多故事支持**：第二个故事（西部风格）开发中

## 3. 技术能力评估

- **底层技术架构**：基于 Agent-Kernel（浙大自研多智能体框架）+ LLM API
- **模型基座**：支持多种 LLM（Qwen-flash 等），通过 API 调用
- **关键技术参数**：
  - 单 Tick token 消耗：约 0.3 元（Qwen-flash）——同类项目中较低
  - 支持 100+ Agent 并发（笔记本级别运行）
  - 依赖 Redis 作为数据总线和缓存
  - Python 3.10+，Ray 分布式运行时
- **技术特色**：
  - ⭐ 动态 Agent 管理（时间换空间架构）让普通硬件也能跑大规模多智能体
  - 完整的 Agent 生命周期插件机制
- **开源情况**：✅ 完全开源（Apache-2.0）
- **API 能力**：提供 HTTP API Server（默认 0.0.0.0:8000）

- **竞争格局简述**：
  - OpenStory 定位独特的**故事推演**赛道，而非通用 Agent 框架
  - 差异化在于 1:1 可视化和动态 Agent 管理，而非通用多代理框架
  - 对比 Stanford小镇等同类项目，token 消耗更低，门槛更亲民

- **差异化定位**：
  > "用 Agent 还原故事，让经典活在眼前"——OpenStory 不只是框架，更是一个让任何人都能参与推演故事的平台

## 10. 总结与展望

- **核心优势**：
  1. **动态 Agent-Kernel 架构**：业界领先的动态增删 Agent 能力，资源效率高
  2. **1:1 可视化还原**：大观园前端质量高，用户体验直观有趣
  3. **低资源消耗**：单 Tick 成本低至 0.3 元，让普通用户也能运行

- **潜在风险**：
  1. **LLM API 成本**：长时间运行推演时 token 消耗可能累积
  2. **学术项目性质**：缺乏商业化支持，长期维护取决于团队研究方向
  3. **生态规模小**：44 stars 与主流框架差距大，社区建设需要时间

- **发展趋势**：
  1. 更多故事内容加入（西部风格等），构建故事内容矩阵
  2. Agent-Kernel 框架可能独立发展，吸引更多开发者
  3. 与游戏/叙事产品结合，探索商业化路径（如 AI NPC 生成）
