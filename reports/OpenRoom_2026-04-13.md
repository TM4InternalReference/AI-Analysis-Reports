## ⚡ 一句话速览

> MiniMax 打造的 AI Agent 浏览器桌面操作系统，把 AI Agent 注入类 macOS 的虚拟桌面空间，用户通过自然语言即可驱动相册、游戏、日记等应用运行，代码大部分由 AI 自生成，是国内 GUI Agent 赛道"空间化交互"路线的代表产品。

## 1. 产品概览

- **产品名称**：OpenRoom
- **所属厂商/公司**：MiniMax（miniMax），中国大模型独角兽公司，成立于2021年，创始人闫俊杰（前商汤科技高管），总部中国
- **产品类型**：AI Agent 平台 / 浏览器桌面操作系统
- **首次上线时间**：2025年（具体日期暂无公开数据）
- **最新版本/代次**：基于 MiniMax M2.7 模型（2026年3月发布）
- **官网地址**：https://openroom.ai / https://github.com/MiniMax-AI/OpenRoom

## 2. 核心功能

按功能模块分类列出，差异化亮点用 ⭐ 标注：

- **虚拟桌面空间**：类 macOS 的浏览器内桌面，Dock 栏+可拖拽窗口+聊天面板，无需安装软件 ⭐
- **AI Agent 感知与操作**：Agent 主动感知用户行为，自然语言驱动 Music、Chess、Gomoku、FreeCell、Email、Diary、Twitter、Album、CyberNews 等9个内置应用 ⭐
- **长期记忆框架**：记忆提取、整合、检索，支持 Agent 记住用户偏好和历史交互 ⭐
- **Vibe Workflow**：自然语言创建新应用，六阶段流水线（需求分析→架构设计→任务规划→代码生成→资源生成→项目集成），深度集成 Claude Code ⭐
- **人设 Agent 系统**：内置 Aoi、Rea、Jill 三个角色，支持自定义 Agent 导入和平台化扩展 ⭐
- **无后端本地存储**：所有数据存于浏览器 IndexedDB，隐私完全由用户掌控 ⭐
- **结构化 Action 系统**：Agent 与应用间通过标准化 Action 通信，降低感知复杂度，提高响应速度 ⭐

## 3. 技术能力评估

- **底层技术架构**：浏览器即桌面（Web Desktop），基于 React 18 + TypeScript + Vite + Tailwind CSS 构建，monorepo 结构（pnpm workspaces + Turborepo）
- **模型基座**：MiniMax 自研 M2.7 多模态大模型（文本+视觉）
- **关键技术参数**：
  - 参数量：M2.7 激活参数约 10B（总参数 230B）
  - 上下文窗口：未公开
  - 多模态支持：支持文本、视觉（GUI 界面理解）、语音
  - 其他：Agent Harness 自进化框架，支持多 Agent 协作
- **Benchmark 表现**：

| 基准测试 | 得分/排名 | 数据来源 |
|---------|----------|---------| 
| SWE-Pro | 56.22%（接近 GPT-5.4 的 57.7%） | 腾讯云/MiniMax 官方 |
| Terminal Bench 2 | 57.0% | MarkTechPost |
| VIBE-Pro（完整项目交付） | 55.6% | MiniMax 官方 |
| GDPval-AA（ELO） | 1495（开源模型最高分） | MiniMax 官方 |
| OpenRouter Token 年化吞吐量 | 连续四周排名第一 | 智源社区 |

- **技术特色**：
  - **Agent Harness 自进化机制**：模型深度参与自身训练与优化，构建复杂 Agent 框架
  - **Action 系统**：结构化感知 vs 视觉感知的混合策略，兼顾效率与通用性
  - **Vibe 变更工作流**：四阶段影响分析→任务规划→代码实现→验证检查，支持对已有应用的自然语言迭代
- **开源情况**：完全开源（MIT 许可协议），代码主要由 AI 自生成
- **API 能力**：主要面向终端用户（Web 界面），MiniMax 开放平台提供 API 支持企业级调用

## 4. 版本迭代

| 版本 | 发布时间 | 主要变化 |
|------|---------|---------| 
| 首发版本 | ~2025年 | 基础桌面框架，9个内置应用，基础 Agent 对话能力 |
| M2.5 集成版 | ~2026年2月 | 接入 M2.5 文本模型，Lightning 版本输出速度 100 TPS |
| M2.7 集成版 | 2026年3月 | Agent Harness 自进化，短时记忆+自反馈+自优化；Vibe Workflow 完善；情商能力增强；M2.7 开源 |

**最新版本关键迭代亮点**（M2.7，相比上一代 M2.5）：
- Agent Harness 自进化机制：模型参与自身训练闭环，不再是静态模型
- SWE-Pro 从 ~40% 提升至 56.22%，跃入顶级模型区间
- 多 Agent 协作支持：Agent 之间可以协同完成复杂任务
- 情商能力专项优化：情感识别、情感回应、情感主动

## 9. 竞品分析

- **所在赛道**：GUI Agent / 浏览器 Agent 桌面 / AI OS
- **主要竞品对比**：

| 维度 | OpenRoom | OpenAI Operator | Claude Computer Use | 智谱 AutoGLM | 字节 UI-TARS |
|------|---------|-----------------|---------------------|--------------|--------------|
| 厂商 | MiniMax | OpenAI | Anthropic | 智谱AI | 字节跳动 |
| 核心卖点 | 浏览器桌面+空间交互 | 云端虚拟机+即开即用 | API 形式+开发者可控 | 自然语言操作手机应用 | GUI 基准测试 SOTA |
| 技术路线 | 结构化 Action + 视觉混合 | 云端虚拟机 | 视觉感知 API | 手机应用操作 | 模型专项训练 |

- **竞争格局简述**：GUI Agent 赛道 2025 年全面爆发，美系（OpenAI、Anthropic）走"通用能力"和"开箱即用"路线；中国系（MiniMax、智谱、字节）各有特色——OpenRoom 的"空间化交互"是独特路线，而非单纯对标 Computer Use。
- **差异化定位**：不是让 AI 控制你的电脑，而是创造一个全新的"数字空间"让 AI 和用户共处——"对话即驱动，空间即界面"

## 10. 总结与展望

- **核心优势**：
  1. **开源+MIT 许可**：业内罕见的完全开源 GUI Agent 桌面平台，降低开发者门槛
  2. **空间化交互范式**：区别于对话框，提供类 macOS 桌面临场感，情感羁绊更深
  3. **AI 自生成代码**：代码大部分由 AI 编写，"元循环"架构代表新型开发范式

- **潜在风险**：
  1. **生态扩展受限**：内置9个应用场景有限，第三方开发者生态尚未形成规模
  2. **商业变现路径不清晰**：OpenRoom 本身免费，主要靠 API 变现，长期商业模式需观察
  3. **安全风险**：GUI Agent 的提示注入攻击（Prompt Injection）是行业共性问题

- **发展趋势**：
  1. 基于 M2.7 自进化能力，Agent 自主性和任务复杂度将持续提升
  2. Vibe Workflow 若能普及，可能成为"自然语言开发"的新标准
  3. 移动端/多端协同预计是下一步扩展方向
  4. 与 MiniMax 上市后资本加持结合，有望加速生态建设
