## 2. 核心功能

按功能模块分类列出，差异化亮点用 ⭐ 标注：

- **AI 端到端游戏生成**：用户输入游戏描述文本，AI 自动完成架构设计、代码编写、资产生成 ⭐
- **Claude Code Skills 协作**：两个 Skill 协同工作——Planner Skill 负责设计规划，Executor Skill 负责执行落地 ⭐
- **Gemini 2D 资产生成**：集成 Google Gemini 生成 2D 美术素材和纹理贴图 ⭐
- **Tripo3D 3D 模型生成**：可选接入 Tripo3D 将 2D 图片转换为 3D 模型 ⭐
- **视觉 QA 自动化**：截取游戏运行截图，用 Gemini Flash 视觉分析检测 Z-fighting、贴图缺失、物理错误等问题 ⭐
- **GDScript 专业支持**：内置 850+ Godot 类别的自定义语言参考和延迟加载 API 文档 ⭐
- **预算感知资产生成**：在生成资产时考虑成本效益，优化每分钱生成的视觉冲击力

## 3. 技术能力评估

- **底层技术架构**：AI Pipeline 架构，两个 Claude Code Skills 协调工作，Python 97.2% / Shell 2.8%
- **模型基座**：Claude Code（Anthropic），主要使用 Opus 4.6；图像生成使用 Google Gemini；3D 模型使用 Tripo3D
- **关键技术参数**：
  - 支持 Godot 4（2D 和 3D）
  - 支持 850+ Godot 类别的 API 文档
  - 兼容 OpenCode 等开源替代方案
- **技术特色**：
  - 视觉 QA 闭环：通过截图 + Gemini Flash 视觉分析实现自动问题检测和修复
  - 预算感知资产生成
  - Context isolation：每个任务在新鲜上下文运行，保持专注
- **开源情况**：✅ 完全开源，MIT 许可证
- **API 能力**：
  - GOOGLE_API_KEY：Gemini 图像生成 + 视觉 QA
  - TRIPO3D_API_KEY：3D 模型生成（仅 3D 游戏需要）

## 9. 竞品分析

- **所在赛道**：AI 游戏开发自动化 / 文本到游戏（Text-to-Game）
- **主要竞品对比**：

| 维度 | Godogen | GameMaker Studio | RPG Maker | Unity + AI |
|------|---------|-----------------|-----------|------------|
| 厂商 | @htdt（个人） | YoYo Games | Enterbrain | Unity Technologies |
| 核心卖点 | AI 端到端生成，Claude Code 驱动 | 2D 游戏专用，无代码/低代码 | RPG 专用，成熟模板生态 | 通用电引擎，AI 辅助（Copilot 等） |
| 技术路线 | LLM + Claude Code Skills | 传统引擎 | 传统引擎 | 传统引擎 + AI |
| 定价 | 免费开源 | $99/年起 | $79.99/永久 | 免费+订阅 |
| 优势 | 零代码、全自动、资产生成 | 2D 专用、成熟稳定 | RPG 模板丰富 | 生态最大、最通用 |
| 劣势 | 依赖 Claude Code，需手动配置 | 仅 2D | 仅 RPG 类型 | 非端到端 AI 生成 |

- **竞争格局简述**：传统游戏引擎（Unity/Unreal/Godot）均在探索 AI 辅助功能，但真正实现"描述即游戏"的端到端方案极少。Godogen 填补了 Claude Code 生态在这一领域的空白。
- **差异化定位**：首个基于 Claude Code Skills 的开源端到端游戏生成工具，强调零人工干预的全自动化管线。

## 10. 总结与展望

- **核心优势**：
  1. 端到端自动化程度高，从描述到可运行游戏
  2. 视觉 QA 闭环确保输出质量
  3. 完全开源，MIT 许可

- **潜在风险**：
  1. 依赖 Claude Code（需付费订阅 Anthropic）
  2. 单人维护，项目可持续性存疑
  3. 生成质量和稳定性尚需大规模验证

- **发展趋势**：
  1. 探索接入更多 AI 模型（如 Grok Image）降低资产生成成本
  2. 支持更多引擎（Roadmap 提到 Bevy）
  3. 发布完整端到端演示 Demo
